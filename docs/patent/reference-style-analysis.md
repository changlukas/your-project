# 參考專利風格分析

**來源：** P51140026-64672 資料傳輸同時運算結構與方法（主管專利）

## 結構模式

1. **要解決什麼問題**：1 句話 + 1 張圖。不做背景教學，直接講痛點。
2. **預計解決方案**：1 句話 + 1 張圖。馬上讓讀者看到解法長什麼樣。
3. **設計背景**：簡短補充技術 context（放在問題和解法之後，不是之前）
4. **專利構想摘要**：1 段核心概念 + 架構特色列表
5. **新穎進步簡述**：1 張 習知 vs 本案 對比表（5 行以內）
6. **實施例**：圖 + 簡短說明，每個實施例 1 頁
7. **前案 PE**：表格，特徵 A/B/C 逐項對比
8. **Claim**：裝置項 + 方法項各一個獨立項，依附項展開

## 關鍵寫作原則

- **問題和解法先行**，背景後補
- **每頁一個概念**，不在同一 section 塞多個概念
- **圖是主角**，文字是圖的說明
- **對比表精簡**，不做 literature review
- **Claim 同時有裝置項和方法項**

## 與我們文件的差距

| 面向 | 主管風格 | 我們目前的問題 |
|---|---|---|
| 問題描述 | 1 句話直擊痛點 | 埋在 2 頁背景之後 |
| 解法描述 | 1 句話 + 1 張圖 | 分散在 8 個子節 |
| 背景 | 極短，放在問題之後 | 1.5 頁，放在最前面 |
| 圖 | 核心溝通工具 | 附帶品，且未更新 |
| 對比 | 1 張精簡表 | 13 種 prior art 的 literature review |
| 敘事順序 | 問題 → 解法 → 背景 → 細節 | 背景 → 背景 → 背景 → 問題 → 解法 |
