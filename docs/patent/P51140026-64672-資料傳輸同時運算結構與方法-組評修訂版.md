# 提案案次/名稱：傳輸中運算 (Process during Transmission, PdT)

# (1)要解決什麼問題?

• 統計相關的運算量不大，但卻需要非常多的資料傳輸頻寬，限制大型AI模型的運算速度。

![](images/2b71641506dafbf94ca9e77a62a170edf24326d8fe62b9c1dad43ecc678a9890.jpg)  
舉例，計算(標準差)需要來回反覆多次存取資料

# (2)預計解決方案

- 記憶體管理器內建數學運算處理，大量節省資料來回存取的次數

![](images/5cfbe63e84389b4195cf8c551cb8f0db65f63bc0b6e5c91911e8378c59fdc023.jpg)  
寫回去資料的過程，就順便計算資料統計資訊

# (3)廠商的回饋(VOC)

# 1. 晶豪科

• 可以有效幫助節約記憶體傳輸量，舒解AI大型模型的頻寬需求限制，可加值記憶體控制器。

# 2. 神盾/芯鼎

• 對於NPU的記憶體使用加速和節能，對效能和功耗有幫助。

# (4)對應的推動規劃

1. 確認產業規格需求

- 確認記憶體的製造工藝和介面型態。

2. 強化與製程廠和IP公司的合作

• 與既有合作的製程廠與其配合的IP公司，透過上架平台，授權給其下單客戶。

# 設計背景：需要讀取重複資料作不同運算

AI運算需要高速、大量的資料搬運，需要高頻寬、大容量的DRAM鄰近運算晶片。  
- 重複資料多的不同運算 (如：統計運算)，需要對同一群資料反覆讀取計算

Find min. from (a, b, c, d, e, f, g...)   
Find max. from (a, b, c, d, e, f, g...)   
Find mean of (a, b, c, d, e, f, g...)   
Find variance of (a, b, c, d, e, f, g...)   
Find root mean square of $(a, b, c, d, e, f, g \ldots)$

→ 本案提出一種架構和方法，讓資料只需要傳輸一次，便能得到多種運算結果。

- 請傳輸資料的過程，同時進行多樣化、常用運算，避免重複讀取資料  
• 對於大型資料的統計相關運算，提高運算效能、降低能源消耗

# 傳輸資料時，同時計算多種函數

• 在傳輸資料的同時側錄(simultaneously recording)，並且計算該數據。  
• 計算包含統計訊息：最大、最小、累加、平均數、變異數，或是線性函數如 $f(x)$ 。  
• 個錄資料行為，不影響記憶體的正常讀寫  
• 透過CPU等處理器動態設定功能、一段連續資料的位置起點與終點

# 專利提案特色

# 新穎性 (Novelty):

1. 傳輸運算同步: 突破傳統先傳輸後運算模式，首創資料傳輸與多樣函數運算同步進行。  
2. 單次傳輸多結果: 資料單次傳輸, 同步產出多種運算結果 (Min, Max, Mean, etc.), 免除重複讀取。  
3. 個錄不干擾: 資料傳輸時「側錄」運算，不影響記憶體正常讀寫，維持系統功能。

# 進步性 (Progressiveness/Advantages):

1. 高效低延遲: 同步運算與免重複讀取，顯著提升效能，降低延遲。  
2. 節省頻寬: 單次傳輸大幅降低記憶體頻寬需求。  
3. 降低功耗: 减少傳輸次數, 降低能源消耗。  
4. 適用連續資料: 專為連續讀寫資料場景優化設計。
